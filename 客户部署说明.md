# MediaCrawler 客户部署说明

本文面向交付客户的本机部署。目标是让客户在自己的电脑、自己的飞书账号和自己的浏览器登录态下运行工作台。

## 一、交付包内容

交付代码目录应包含：

- `start_ops.bat`：Windows 启动入口。
- `start_ops.sh`：macOS / Linux 启动入口。
- `api/`、`config/`、`media_platform/`、`store/`、`tools/` 等项目代码。
- `requirements.txt`：Python 依赖清单。
- `api/webui/ops_config.html`：客户工作台页面。

交付包不应包含：

- `.venv/`
- `.git/`
- 本机 `data/`、`downloads/`、`browser_data/`
- 任何客户 Cookie、飞书 Token、历史项目配置

运行后的客户配置、数据和浏览器登录态默认保存在客户系统用户目录下，不需要放在代码目录里。

## 二、客户电脑前置要求

Windows 和 macOS 都可以部署，前提如下：

- Python 3.11 或更高版本。
- Chrome 或 Edge 浏览器。
- 可访问 Python 包安装源。
- 飞书 CLI：命令行里能执行 `lark-cli`。
- `lark-cli` 已用客户自己的飞书账号完成授权。

飞书相关能力依赖 `lark-cli`：

- 一键初始化项目。
- 复制母版 Base。
- 读取表结构。
- 写入多维表记录。
- 上传附件。

如果客户只做本地抓取、不写入飞书，可以暂时不配置 `lark-cli`；但完整交付流程必须配置。

## 三、母版 Base 设置

当前项目初始化逻辑是“复制母版 Base”，不是从代码里凭空创建完整模板。

工作台步骤 1 需要填写：

- 所属项目名。
- 母版 Base 链接 / Token。

后端会调用：

```text
lark-cli base +base-copy --as user --base-token <母版Token> --name <项目名> --without-content
```

也就是说：复制母版结构，不复制母版数据。

母版 Base 至少应包含这些表：

- `账号筛选表`
- `爆款监控表`
- `笔记二创表`
- `笔记评论表`
- `合作笔记监控表`
- `达人智能圈选表`

推荐交付方式：

- 交付方维护一份标准母版 Base。
- 客户把标准母版复制到自己的飞书空间，或交付方把母版共享给客户并授予复制/编辑权限。
- 客户在工作台填入客户自己可访问的母版链接。

## 四、首次部署流程

### Windows

1. 解压交付包到客户电脑，例如 `D:\MediaCrawler`。
2. 双击 `start_ops.bat`。
3. 脚本会自动创建 `.venv`、安装 Python 依赖、安装 Playwright Chromium。
4. 浏览器会打开：

```text
http://127.0.0.1:8081/ops-config
```

### macOS / Linux

1. 解压交付包到客户电脑。
2. 在终端进入项目目录。
3. 执行：

```bash
chmod +x start_ops.sh
./start_ops.sh
```

4. 脚本会自动创建 `.venv`、安装 Python 依赖、安装 Playwright Chromium。
5. 浏览器会打开：

```text
http://127.0.0.1:8081/ops-config
```

## 五、首次使用流程

1. 打开工作台后，在“项目管理”里点击“检测当前电脑环境”。
2. 确认 Python、依赖、Playwright 浏览器通过。
3. 确认 `lark-cli` 已安装，并已用客户飞书账号授权。
4. 填写客户项目名。
5. 填写客户母版 Base 链接 / Token。
6. 点击“一键初始化项目（复制母版）”。
7. 到“公共配置”确认登录方式，建议优先使用扫码登录。
8. 到对应模块执行样本账号抓取、达人智能圈选、爆款内容检索和多维表同步。

## 六、后续升级

后续新增或改变功能时，应区分代码和客户运行数据：

- 可以替换项目代码目录。
- 不要删除客户用户目录下的运行数据。
- 不要覆盖客户已保存的 Cookie、浏览器登录态、项目配置。

如果只是代码修复：

1. 停止当前工作台窗口。
2. 替换代码目录。
3. 重新运行 `start_ops.bat` 或 `start_ops.sh`。

如果新增飞书字段或表结构：

- 新客户：使用新版母版 Base 初始化。
- 老客户：需要在已有 Base 里补字段，或新建项目并迁移数据。

## 七、常见问题

### 找不到 `lark-cli`

现象：初始化项目或同步飞书时报“未找到 lark-cli”。

处理：

- 安装飞书 CLI。
- 确认命令行里能执行 `lark-cli`。
- 用客户飞书账号完成授权。
- 重启工作台。

### Playwright 浏览器不可用

现象：扫码登录、蒲公英自动化或抓取启动失败。

处理：

```bash
python -m playwright install chromium
```

如果使用虚拟环境，请使用 `.venv` 里的 Python 执行。

### 端口 8081 被占用

Windows 可修改 `start_ops.bat`：

```bat
set "PORT=8082"
```

macOS / Linux 可启动时指定：

```bash
PORT=8082 ./start_ops.sh
```

### 工作台带出旧项目或旧 Cookie

在“项目管理”里点击“清空工作台配置”。清空后重新填写客户项目名和母版 Base。

### 母版复制成功但没有回填表 ID

通常是母版里的表名与系统预期不一致。请检查表名是否包含：

- 账号筛选
- 爆款
- 二创
- 评论
- 合作
- 达人智能圈选
