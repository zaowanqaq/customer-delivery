# 艾莉芬特 (Aliafant) UI 升级与重构计划

## 1. 当前状态分析
- **UI 现状**：目前系统的操作界面集中在 `api/webui/ops_config.html`，采用简单的上下滑动表单流布局，内联 CSS，视觉风格偏向基础的开发测试工具。
- **业务诉求**：因项目需要对外向客户（艾莉芬特）展示，需将界面升级为符合其“专业全域内容营销服务商”定位的现代化、高品质 SaaS 管理后台样式。
- **代码状态**：当前工作区存在未提交的修改，需先安全保存，并在隔离的 Git 分支中进行 UI 迭代。

## 2. 拟定更改方案

### 第一步：Git 版本控制操作
- **保存现状**：执行 `git add .` 与 `git commit -m "chore: save previous modifications"` 保存当前所有修改。
- **创建新分支**：执行 `git checkout -b feature/aliafant-ui-redesign` 新建专门用于 UI 重构的分支，以便您后续决定是否合并。

### 第二步：界面布局与结构重构 (SaaS 侧边栏模式)
修改目标文件：`api/webui/ops_config.html`
- **整体框架**：废弃单页垂直流，改为 **左侧固定导航栏 (Sidebar) + 右侧动态内容区 (Main Content)** 的经典 SaaS 布局。
- **左侧导航栏 (Sidebar)**：
  - **顶部品牌区**：展示“ALIAFANT 艾莉芬特”品牌标识与副标题“全域营销数据中台”。
  - **菜单项设计**：按业务工作流划分为 5 个独立 Tab：
    1. 项目管理 (初始化与切换)
    2. 样本账号抓取 (步骤 2)
    3. 爆款内容抓取 (步骤 3)
    4. 数据同步中心 (步骤 4)
    5. 合作与运行监控 (步骤 5 + 终端日志)
- **右侧内容区 (Main Content)**：
  - 采用选项卡式 (Tab) 切换逻辑，点击左侧菜单时，右侧仅显示对应的功能卡片，减轻视觉负担。

### 第三步：视觉风格与体验优化 (高级感与现代设计)
- **色彩规范 (Color Palette)**：
  - 借鉴 Landingfolio 与优秀 SaaS 官网风格，采用极简的“黑/白/灰”作为主基调，凸显专业感。
  - 强调色 (Primary Color) 采用高级的小红书红 (如 `#E02424` 或深邃红)，用于按钮、Active 状态和关键高亮。
- **排版与间距 (Typography & Spacing)**：
  - 增加留白 (Padding/Margin)，使用系统级现代无衬线字体 (system-ui, -apple-system)。
  - 统一输入框、按钮的高度和交互反馈 (Hover 动效、Focus 边框高亮)。
- **组件质感 (Components)**：
  - 卡片化设计：内容区使用带柔和阴影 (`box-shadow: 0 4px 20px rgba(0,0,0,0.05)`) 和圆角 (`border-radius: 12px`) 的白色卡片承载表单。
  - 美化现有的终端日志框 (Monitor Log)，使其更像极客风的专业控制台。

### 第四步：交互逻辑重构 (JS)
- 在现有的 `<script>` 中补充选项卡切换逻辑，控制 DOM 元素的 `display` 状态。
- 确保所有的现有表单 ID 和数据绑定逻辑不受影响，仅改变其 DOM 层级结构。

## 3. 假设与决策
- **技术栈限制**：为了不破坏现有的部署方式，UI 重构将继续在 `ops_config.html` 中通过原生 HTML/CSS/JS 实现，不引入重量级的外部前端框架（如 React/Vue 或完整版 Tailwind），而是手写高质量的现代 CSS。
- **图标引入**：将引入轻量级的 SVG 图标（或通过 CDN 引入 FontAwesome 等）以美化侧边栏。

## 4. 验证步骤
1. 完成 Git 操作后，通过 `git status` 和 `git branch` 确认分支状态正确。
2. 完成 HTML 修改后，启动 FastAPI 服务，访问本地端口。
3. 验证左侧导航栏点击是否能正确切换右侧面板。
4. 验证各个表单步骤（初始化、抓取、同步）的按钮点击和 API 请求是否正常工作，确保 UI 修改未破坏原有业务逻辑。
